# 项目变更单（计划变更一）

## 1. 基本信息
- 变更单编号：CR-2026-001
- 项目名称：云南省企业就业失业数据采集系统
- 变更主题：企业填报周期由“全年月报”调整为“1-3月半月报、4-12月整月报”
- 提出日期：2026-04-23
- 申请部门：业务管理部门
- 变更级别：重要需求变更（影响业务规则、默认周期配置、界面文案、验收口径）

## 2. 变更背景
原规则为企业按月填报。根据最新业务要求：
1. 每年1月、2月、3月实行半月报（上半月、下半月各一次）。
2. 每年4月至12月实行整月报（每月一次）。
3. 需形成正式变更单并随项目资料上交。

## 3. 变更内容
### 3.1 业务规则变更
- 调查期规则调整为：
  - 1-3月：仅允许两类调查期区间
    - 上半月：1日-15日
    - 下半月：16日-月末
  - 4-12月：仅允许整月区间
    - 1日-月末

### 3.2 系统实现变更
- 默认种子调查期改为：
  - 1-3月自动生成上半月、下半月两期
  - 4-12月自动生成整月一期
- 调查期保存时新增规则校验：
  - 不允许跨月
  - 不符合上述区间规则时拒绝保存并提示错误
- 示例报表与通知文案同步为新规则
- 首页与调查期管理文案同步新规则说明

## 4. 影响分析
### 4.1 影响范围
- 后端：调查期生成与校验逻辑、示例数据
- 前端：页面说明文字、调查期示例提示
- 测试：调查期配置与填报流程回归测试

### 4.2 兼容性评估
- 数据结构不变（无新增表字段）
- 接口结构不变（前后端交互字段保持兼容）
- 老数据可继续使用，新增规则主要作用于新建/维护调查期

### 4.3 风险与应对
- 风险：省级用户手工维护调查期时配置不符合规则
- 应对：系统端校验拦截并返回明确错误信息

## 5. 计划与里程碑
- D1：完成后端周期规则与默认周期生成改造
- D1：完成前端文案与提示更新
- D2：完成联调与回归测试（含1-3月半月场景、4-12月整月场景）
- D2：提交变更材料归档

## 6. 验收标准
1. 1-3月可配置并使用上半月/下半月两期；禁止配置为整月一期。
2. 4-12月仅允许整月一期；禁止配置半月区间。
3. 企业在有效调查期窗口内可正常填报、上报、审核。
4. 页面文案体现“1-3月半月报、4-12月整月报”。
5. 变更单可作为项目交付文档归档上交。
