# 项目变更单（计划变更二）

## 1. 基本信息
- 变更单编号：CR-2026-002
- 项目名称：云南省企业就业失业数据采集系统
- 变更主题：企业上报与市/省审核能力支持手机端操作，并调整项目工期
- 提出日期：2026-04-10
- 申请部门：业务管理部门
- 变更级别：重要需求变更（涉及前端交互、测试范围、项目计划）

## 2. 变更背景
当前系统核心流程主要按桌面端交互设计。根据业务现场使用反馈，需要在手机端完成以下关键流程：
1. 企业数据上报（含保存草稿、提交报表）
2. 市级审核（单条审核、批量审核）
3. 省级审核（审核、修正、上报相关操作）

同时，因新增移动端适配、联调与回归测试工作，项目计划需增加两周。

## 3. 变更内容
### 3.1 功能范围变更
- 新增要求：关键业务流程必须支持手机端完成，不仅可浏览。
- 手机端必达场景：
  - 企业端：备案与报表保存/提交
  - 市级端：待审数据加载、通过/退回、批量审核
  - 省级端：审核、修正、上报相关关键按钮操作

### 3.2 交互与界面变更
- 对关键操作区进行移动端优化：
  - 小屏下按钮全宽显示，提升点击命中率
  - 核心操作区在手机端采用粘性操作条，便于连续处理
  - 页面网格在小屏设备自动折叠为单列布局
  - 统计图表区域在移动端单列展示

### 3.3 工期变更
- 原计划总工期：2个月
- 变更后总工期：2个月 + 2周
- 增量工期：2周

## 4. 影响分析
### 4.1 影响范围
- 前端：响应式布局、关键操作区交互优化
- 测试：新增移动端兼容与可用性回归
- 项目管理：里程碑与交付日期顺延两周

### 4.2 风险与应对
- 风险：移动端按钮密集导致误触
  - 应对：关键按钮全宽、大尺寸触控区、分组展示
- 风险：不同机型显示差异
  - 应对：按主流分辨率进行兼容测试（Android/iOS）

## 5. 计划与里程碑调整
- M1（原）：核心功能开发完成
- M2（原）：联调与验收准备
- M2+2周（新）：
  - 移动端适配完善
  - 移动端回归测试完成
  - 变更验收与文档归档

## 6. 验收标准
1. 在手机浏览器中，企业上报流程可完整执行（保存/提交可用）。
2. 在手机浏览器中，市级审核关键按钮与批量审核功能可执行。
3. 在手机浏览器中，省级审核与修正关键功能可执行。
4. 页面在小屏设备无关键内容溢出、按钮可点击、流程可达。
5. 项目总工期按“2个月+2周”更新并通过审批。
